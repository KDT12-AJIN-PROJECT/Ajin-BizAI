# 2026-05-30 작업 기록

## 1. Docker 백엔드 연결 구성 (신규)

**문제:** 프론트엔드가 `localhost:8000`(하드코딩)으로 요청 → Docker 환경에서 백엔드 없음 → 공고 미표시

**수정:**
- `backend/Dockerfile` 신규 생성 (Python 3.13-slim, uvicorn)
- `backend/.env` 신규 생성 (`.env.server` 변수명 매핑: `API_KEY`→`GONGGONG_API_KEY`, `BIZ_KEY`→`BIZINFO_API_KEY`)
- `backend/database.py` SQLite 경로 `./ajin.db` → `./data/ajin.db` (Docker named volume 영속성)
- `web-react/docker-compose.yml` backend 서비스 + `ajin_db` 볼륨 추가
- `web-react/nginx.conf` `/api/` → `http://backend:8000` 프록시 추가
- `web-react/Dockerfile` `VITE_API_BASE_URL=""` build arg 추가 → 빌드 시 상대경로 사용

## 2. LM Studio AI 기능 복구

**문제:** `POST /proxy/lmstudio/...` → nginx 405 → AI 분석 전체 불동작 (공고 요약, 사업계획서 작성 등)

**수정:**
- `web-react/nginx.conf`에 `/proxy/lmstudio/` → `http://host.docker.internal:1234/` 프록시 추가 (timeout 120s)

**결과:** 405 → 401 (LM Studio 인증 정상 도달)

## 3. 공고 API 응답 크기 축소

**문제:** `/api/notices/search` 매 로드마다 3.1MB 전체 반환 → nginx temp file 버퍼링 경고 반복

**수정:**
- `backend/routers/notices.py` `search_notices` + `get_notices` 에 `limit` 파라미터 추가 (기본 500, 최대 1000)
- `web-react/nginx.conf` gzip 압축 + proxy buffer 설정 추가

**결과:** 응답 3.1MB → 1.2MB

## 4. 북마크 전체 삭제 백엔드 연동

**문제:** `clearBookmarks()` 프론트 localStorage만 비우고 DB는 그대로 → 재접속 시 북마크 복원

**수정:**
- `backend/routers/bookmarks.py` `DELETE /api/bookmarks` (전체 삭제) 엔드포인트 추가
- `web-react/src/api/backendApi.js` `bookmarksApi.clearAll()` 추가
- `web-react/src/features/notices/hooks/useBookmarks.js` `clearBookmarks`에서 `clearAll()` 호출

## 5. evaluate_draft JSON 파싱 구현

**문제:** `local_provider.py evaluate_draft` 가 LLM 응답 파싱 없이 `score: 0, grade: "?"` 고정 반환

**수정:**
- `backend/services/local_provider.py` LLM 응답에서 JSON 추출 (`re.search + json.loads`)
- 파싱 실패 시 기존 fallback 유지

## 6. Step 3 초안 작성 DB 저장 미구현 수정

**문제 진단:**
- `write-draft-item` / `rewrite-draft-item` / `approve-draft-item` 세 엔드포인트 모두 DB 저장 없이 응답만 반환
- 생성된 초안이 React state에만 존재 → 새로고침 시 전부 사라짐
- `getDraftItems`는 `form_schema_json.draft_items`만 읽는데, 생성된 content가 없음
- 주석 `"Phase 4-E mock — DB DraftItem 저장은 Phase 4-G"` 미완 상태 그대로

**수정:**
- `backend/routers/analysis.py` — `_update_draft_item_in_fsj()` 헬퍼 함수 추가
  - `form_schema_json.draft_items`에서 `question_id` 매칭 → updates 적용 → `flag_modified` → commit
  - 항목 없으면 최소 레코드 신규 생성
- `write-draft-item` — 생성된 `content`, `matched_evidence_ids`, `table_draft_data` 저장
- `rewrite-draft-item` — 재작성된 `draft_text` 저장
- `approve-draft-item` — `status=approved`, `approved_at` 저장

**추가 진단 (미수정):**
- `AI_PROVIDER` 기본값 `mock` → mapping pipeline 전체가 가짜 데이터 반환
  - `openai`/`anthropic` 사용하려면 `backend/.env`에 API 키 + `AI_PROVIDER=openai` 설정 필요
  - `local` (LM Studio)는 v0.2 mapping 모듈이 미구현 stub → mock 위임 유지
- `backend/.env`에 `AI_PROVIDER` 주석 + 기본값 `mock` 명시 추가

## 7. Azure OpenAI 연동 (모든 AI → o4-mini)

**설정:**
- `backend/.env` Azure 크레덴셜 추가
  - `AZURE_OPENAI_ENDPOINT=https://ajin-project.cognitiveservices.azure.com/`
  - `AZURE_OPENAI_API_KEY=...`
  - `AZURE_OPENAI_API_VERSION=2024-12-01-preview`
  - `AZURE_OPENAI_DEPLOYMENT=o4-mini`
  - `AI_PROVIDER=azure`

**신규 파일:**
- `backend/services/azure_openai_provider.py` 생성
  - `OpenAIProvider` 상속 → `_get_client` + `_chat` + `draft_rewriter` override
  - `AsyncAzureOpenAI` 클라이언트 사용
  - o4-mini reasoning 모델 대응:
    - `max_tokens` → `max_completion_tokens=max(max_tokens*4, 16000)` (reasoning 토큰 여분 확보)
    - `temperature`/`top_p` 제외 (reasoning 모델 미지원)
    - `system` role → `developer` role 변환
  - `draft_rewriter` 실제 Azure 호출 구현 (OpenAIProvider mock 위임 override)
- `backend/services/ai_provider.py` `get_provider()` 에 `azure` 케이스 추가

**비용 추정 (o4-mini):**
- 입력 ~$1.10/M, reasoning+출력 ~$4.40/M
- mapping pipeline 1회 ≈ 50k 토큰 ≈ $0.22
- $100 기준 ≈ 450 세션 가능

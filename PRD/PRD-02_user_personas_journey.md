### 2.1 기본 프로필

| 항목 | 내용 |
|------|------|
| **이름** | 박성민 (가명) |
| **직급** | 차장 (기획팀장) |
| **IT 숙련도** | 보고서 받는 용도, 직접 조작 최소화 선호 |
| **의사결정 기준** | 선정 가능성 %, 지원 금액, 준비 기간 |

### 2.2 팀장의 핵심 니즈

1. **"이 공고 우리가 될 것 같아?"** → AI 선정 가능성 점수 즉시 확인
2. **"얼마나 받을 수 있어?"** → 지원 금액·비율 요약 카드
3. **"언제까지 내면 돼?"** → D-Day 표시
4. **"이거 우리가 하는 사업이랑 겹치는 거 아니야?"** → 동시수행 제한 경고

## 4. Error Path — 실패 시나리오

### 4.1 LLM 미연결 시나리오

```
발생 조건: LM_STUDIO_URL 서버가 응답 없음 (연결 실패)

1. ChatDraftPage에서 "공고 분석 시작" 클릭
2. fetch 실패 → 에러 캐치
3. UI: 채팅 패널에 빨간 배너 표시
   메시지: "AI 연결에 실패했습니다. LM Studio가 실행 중인지 확인하거나,
            .env 파일에 OPENAI_API_KEY를 설정하세요."
4. 버튼 표시: [재시도] [빠른 초안으로 전환]
5. "빠른 초안으로 전환" 클릭 → QuickDraftPage로 이동 (기본 템플릿 사용)
```

### 4.2 공공 API 전체 실패 시나리오

```
발생 조건: 4개 API 모두 응답 실패 (네트워크 오류 또는 키 만료)

1. 공고 검색 페이지 접속
2. Promise.allSettled → 모두 rejected
3. notices = [] → Mock 데이터 로드
4. UI: 상단 주황색 경고 배너
   메시지: "외부 공고 API 연결에 실패하여 샘플 데이터를 표시합니다.
            API 키 설정을 확인하세요. [설정으로 이동]"
5. Mock 공고 3건 표시 (정상 작동처럼 보임)
```

### 4.3 파일 파싱 실패 시나리오

```
발생 조건: HWP 파일 업로드 시 pdfplumber 파싱 불가

1. ApplyPrepPage STEP 1에서 HWP 파일 업로드
2. 백엔드 /api/parse-file → 파싱 실패 응답 반환
3. UI: 해당 파일 행에 노란색 경고 아이콘
   메시지: "신청서_양식.hwp 파일을 자동으로 읽지 못했습니다.
            내용을 직접 텍스트로 붙여넣으실 수 있습니다."
4. "텍스트로 직접 입력" 버튼 → 텍스트 입력 모달 표시
5. 사용자가 텍스트 입력 → 분석에 포함
```

## 5. UX 원칙 (5가지)

### 원칙 1: 역방향 정보 수집 (Reverse Data Collection)

> "기업 정보를 주세요" ❌ → "이 정보가 필요한데 혹시 갖고 계신가요?" ✅

```jsx
// 나쁜 예 (지양)
<p>기업의 매출액, 직원 수, 불량률을 입력해주세요.</p>

// 좋은 예 (지향)
<p>
  이 공고의 '기대효과' 항목에서 <strong>현재 공정 불량률</strong> 수치를
  요구합니다. 현재 불량률이 어떻게 되나요?
</p>
<p className="text-xs text-muted-foreground">
  없으시면 건너뛰어도 됩니다. 단, 해당 항목 점수가 낮아질 수 있습니다.
</p>
```

### 원칙 2: 진행 상황 항상 가시화 (Always Visible Progress)

```jsx
// 모든 긴 작업에는 진행 표시기 필수
// 예: AI 분석 중
<div className="flex items-center gap-2 text-sm text-muted-foreground">
  <Loader2 className="w-4 h-4 animate-spin" />
  <span>공고문을 분석하고 있습니다... (약 10초)</span>
</div>
```

### 원칙 3: 모든 AI 결과는 편집 가능 (AI Output Is Always Editable)

- AI가 생성한 초안은 텍스트 에디터에서 직접 수정 가능
- AI 평가 점수는 "추정치"임을 명시, 원문 링크 제공
- 체크리스트는 사용자가 수동으로도 체크/언체크 가능

### 6.2 키보드 네비게이션

```jsx
// 모달 포커스 트랩 구현 예시
useEffect(() => {
  if (!isOpen) return
  const focusable = modalRef.current.querySelectorAll(
    'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
  )
  const first = focusable[0]
  const last = focusable[focusable.length - 1]
  first?.focus()
  const handleTab = (e) => {
    if (e.key !== 'Tab') return
    if (e.shiftKey && document.activeElement === first) {
      e.preventDefault(); last.focus()
    } else if (!e.shiftKey && document.activeElement === last) {
      e.preventDefault(); first.focus()
    }
  }
  document.addEventListener('keydown', handleTab)
  return () => document.removeEventListener('keydown', handleTab)
}, [isOpen])
```
